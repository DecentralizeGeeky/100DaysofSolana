## 100-Day Roadmap

This is a day-by-day path mapped to the numbered folders. Each day ships a small, real outcome.

### 01-initiation (Days 01-10)

1. Day 01: Install toolchain, verify versions, run a hello script.
2. Day 02: Configure wallet, airdrop devnet SOL, confirm balance.
3. Day 03: Send a native SOL transfer and inspect the transaction.
4. Day 04: Read account data with RPC and decode basic fields.
5. Day 05: Create a keypair, sign a message, verify signature.
6. Day 06: Build a minimal program, deploy, and call one instruction.
7. Day 07: Add instruction data parsing and custom error codes.
8. Day 08: Write a basic client that creates and reads a program account.
9. Day 09: Add tests for the program using local validator.
10. Day 10: Document your first program and publish a short demo.

### 02-ascension (Days 11-25)

11. Day 11: Define account layouts and size calculations.
12. Day 12: Implement account initialization with rent exemption.
13. Day 13: Add authority checks and signer validation.
14. Day 14: Introduce PDA derivation and verify seeds.
15. Day 15: Create and update a PDA-owned state account.
16. Day 16: Add versioning to account data for upgrades.
17. Day 17: Implement safe close and reclaim lamports.
18. Day 18: Emit program logs for key state changes.
19. Day 19: Build client helpers for PDA discovery.
20. Day 20: Add tests for PDA edge cases and invalid seeds.
21. Day 21: Explore CPI basics with a simple token transfer.
22. Day 22: Add account constraints and validation helpers.
23. Day 23: Benchmark compute units and optimize hot paths.
24. Day 24: Create a lightweight program spec in README.
25. Day 25: Ship a mini project using PDAs end to end.

### 03-systems (Days 26-45)

26. Day 26: Design a simple escrow state machine.
27. Day 27: Implement escrow init and deposit flow.
28. Day 28: Implement escrow settle and cancel flow.
29. Day 29: Write tests for escrow happy and failure paths.
30. Day 30: Add event logs and client parsing.
31. Day 31: Design a marketplace listing model.
32. Day 32: Implement list and delist instructions.
33. Day 33: Implement purchase flow with fees.
34. Day 34: Add marketplace tests with multiple actors.
35. Day 35: Add identity registry with simple claims.
36. Day 36: Implement claim creation and verification checks.
37. Day 37: Add revocation flow and audit trail.
38. Day 38: Write client SDK helpers for the registry.
39. Day 39: Add rate limits or throttling for spam control.
40. Day 40: Threat model for the systems you built.
